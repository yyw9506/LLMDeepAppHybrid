在沙盘派单功能中，一共有两种角色，派单人和代理派单人。
1.代理派单人是实现对其关联的任一个人员角色是派单人的所有下级执行人下发任务。
2.人员角色为代理派单人是在【代理派单人维护】页面进行人员维护。
3.人员角色为派单人是在【执行人维护】页面进行人员维护。
4.代理派单人是在派单人角色的编辑页面内由字段“代理派单人”进行关联。
5.代理派单人的区域要求是派单人区域的同级或者下级区域，则派单人可成功关联该代理派单人。
6.代理派单人账号在派单时可成功查询到自己、所关联的派单人及下级执行人。
7.代理派单人若派单时只能查询到自己，则确认代理派单人是否在派单人编辑页面字段“代理派单人”内选中。
8.代理派单人若派单时只能查询到自己和派单人，则确认派单人是否和下级执行人成功关联。
9.派单人和下级执行人通过字段“上级oa号、上级cb号”绑定，需确认派单人“人员oa号、人员cb号”和下级执行人的“上级oa号、上级cb号”是一致的。
10.代理派单人和派单人提示“该角色不是小ceo”，需确认人员维护时字段“是否小CEO”是选择“是”。

沙盘看数作战单元、作战结果版块，指标数据是由“经分”系统加工，沙盘结合“用户中心”系统，通过网格id关联，实现经分指标上沙盘。
1. 作战单元、作战结果的，月指标数据每个月10号具备，日数据每天8.30具备。
2. 作战单元的ceo的信息，是通过“用户中心”系统的组织基本信息中的“小ceo”字段确定的。
3. 作战单元的经营发展指标，除了收入和毛利，其他所有指标都是经分直接计算好的。
4. 智家工程师是通过划小接口的“智家工程师月表”判断的。
5. 作战单元、作战结果版块的数据最小层级只到网格层级。

沙盘看数作战地图版块，每天更新，具备时间为10:30。
1. 作战地图的四个模块，社区、农村、校园、聚类，都是通过地理空间数据和资源数据的整合，完成指标上图的。
2. 社区模块的指标，“智网”系统提供的小区数据、用户数据。沙盘利用空间能力——面面匹配，建立小区和网格的对应关系。
3. 农村版块的指标，“智网”系统提供的农村数据、用户常驻经纬度。沙盘利用空间能力——点面匹配，建立用户和农村的关系、乡镇和网格的关系。农村和乡镇通过编码关联。
4. 校园版块的指标，“智网”系统提供的校园数据、用户常驻经纬度。沙盘利用空间能力——实时圈选能力，用当前层级的围栏圈选校园，获取各类指标。
5. 聚类版块的指标，“云商平台”系统提供商户数据。沙盘利用空间能力——点面匹配，建立资源和商户的关系。

如何建立自定义模型派单。
1.“我的工作台”的【自定义模型管理】页面是实现“自定义模型”创建、上架、下架等操作的管理页面。
2.自定义模型可以实现由自助服务平台推送、数据开发治理平台订阅、自定义文件导入三种方式创建。
3.模型创建时上架到“作战行动-用户精准营销-网络小区营销派单”可以用于模型派单。
4.模型派单前要求在【沙盘策略配置】成功绑定来源于全客的自动策略或手动策略。
5.自动策略不需要派单人关联执行人，是基于渠道下发任务。
6.手动策略需要派单人关联执行人，是按人员平均分配原则下发任务。
7.点击任务派发时，提示“该模型未绑定策略，请前往我的工作台-沙盘管理-沙盘策略配置绑定策略”，则确认该模型是否成功绑定策略。
8.点击任务派发时，提示“该角色执行人界面信息录入有误，请前往我的工作台-沙盘管理-执行人维护页面修正”，则确认该派单人是否成功关联执行人。
9.自定义模型成功绑定策略后，派单人在模型达到失效时间前可以成功下发任务。
10.模型达到失效时间后，会自动解绑策略并在该省菜单”作战行动-网络小区营销派单“成功下架该模型。

通用知识。
1. 沙盘的地图坐标系是用的百度09坐标系。
2. 沙盘的提数需求，数据支持需求，请联系方案-李悦。
3. 渗透率小区明细，有存在的数据资产ads_community_3sources_df_pt_rc，直接订购就行。
4. 沙盘的网格数据每天都从用户中心同步，省分上传围栏稽核通过就会更新上图，稽核不通过的话就还是用上一次稽核通过的数据上图。
5. 沙盘上宽带用户和分光器都是智网直接提供的和小区的对应关系，两者之间并没有直接的加工逻辑。宽带用是户是省分在号线系统里维护的用户对应的标准7级地址，然后智网采集的。
6. 出现用户正在点面匹配过程中，请重试，这个问题是在计算上图和未上图过程中数据量太大，需全客侧处理。
